Helping tax filers get USAA tax info into TurboTax

Product designer (Platform)
November 2017
Platforms
  • Web
  • Mobile web
Team

Platform: product designer, content designer, product manager, architect, 3 engineers, 2 business dev, USAA rep

TurboTax: product designer, product manager, engineer lead

Context

I led the experience design of a tax form import widget for TurboTax Online (TTO). This integration allows users to automatically import data from tax forms into TurboTax and complete their taxes accurately and efficiently.

This case study focuses on one use case from the larger effort. For confidentiality reasons, various details are ommitted.

Problem

USAA has a low tax form import success rate in TurboTax

USAA customers struggle to import their tax data from USAA into TurboTax because of unclear sign-in instructions. The goal is to improve the success rate of this task.

User impact

TurboTax users unable to automatically import their data have to enter tax info manually, causing undue frustration with an inherently stressful task. It's time consuming and lowers users’ confidence in having correctly filed their taxes.

Business impact

Getting accurate financial data is key to TurboTax's competitive market advantage. A seamless import experience lowers customer support calls and strengthens Intuit’s partnership with USAA.

Process

  1. Understand current experience
  2. Identify pain points
  3. Brainstorm ideas
  4. Test ideas, gain insights and iterate
  5. Final design documentaiton

TurboTax's initial experience

In TurboTax, most banks request username and password to authenticate and import data into the product. Unlike these banks, USAA asks filers for username and a special access code, which can only obtained via a unique link to USAA’s bank site.

Usual authentication method

USAA's authentication method

Identifying pain points

I focused on how TurboTax was instructing users to get their access code and had several hypotheses with where users were running into trouble. I prototyped the control experience in Axure ran usability tests with 10 US users who were customers of both USAA and TurboTax. Test bank credentials were provided to the participants.

Research insights

  1. No one succeeded on the first try, all users tried their password instead of code
  2. Users focus on authentication fields and don't read instructions
  3. No one saw the link to USAA's site in the legal text on the first try

Defining principles

Tax filers need clear, noticeable instructions on how to import their data from USAA into TurboTax so that they can file taxes properly and maximize their tax returns.

With the research insights, I worked with the product manager to define the following principles:

  1. Set the right expectations
  2. Specific error design
  3. Scalable to similar use cases

Ideation

I brainstormed ideas and narrowed on 2 ideas to test based on prioritizing user benefit, team feedback, project timelines, current product patterns, and contractual agreements with USAA.

Version A 2-step flow

Hypothesis: If we separate the instructions from the authentication fields, users can focus on getting the code first then authentication, rather than both at once.

Axure prototype

Version B 3-step flow

Hypothesis: If we separate the authentication instructions into individual steps, users will pay more attention because the experience differs noticeably from the usual paradigm.

Axure prototype

Testing

I ran a usability test on UserTesting.com.

  • Unmoderated task completion
  • 5 USAA/TurboTax customer per condition
  • Measured for success rate, error recovery rate, time on task, and user comprehension

Research insights

3-step flow was more successful than 2-step flow

  • 3-step flow had 2x higher success rate on the first try
  • Less than 50% of users read instructions in the 2-step flow
  • 100% of users read instructions in the 3-step flow

I shared the research and initial recommendation to stakeholders on our platform team and TurboTax’s team.

Further ideation

Despite the success of the 3-step flow over the 2-step flow, there were still usability issues.

  1. Instructional issues: some users didn't know to click the link
  2. Discoverability issues: some users didn’t see the link

Version A
Use a prominent button

Hypothesis: If we use a prominent call-to-action button in the body, users will have higher import success rate because they clearly see and know how to go to USAA.

Version B
Use the primary button

Hypothesis: If we use the primary call-to-action button in the navigation footer, users will have higher import success rate because they can directly navigate forward to USAA in the flow.

Testing

I ran a usability test on UserTesting.com.

  • Unmoderated task completion
  • 5 USAA/TurboTax customer per condition
  • Measured for success rate, error recovery rate, time on task, and user comprehension

Research insights

Version B was more successful for users
because it was linear and consistent with existing patterns users were accustomed to in TurboTax. Also, using a verb in the primary button made it clear to users what was next.

Final deliverable

  • Interaction, visual and accessibility design spec in Zeplin
  • Content design
  • Customization options and guidlines
  • Analytics
  • Design recommendations for USAA's site

Other considerations

  • Partnering with USAA to provide a seamless hand-off between TurboTax and USAA as the user moves between the two
  • Collaborating with backend architects to ensure this customized solution for USAA scales to future similar use cases
  • Engaging with TurboTax designers to understand TurboTax patterns while ensuring the design scales to all Intuit products
  • Working with legal and compliance teams within Intuit and with partners

Impact

  • Increased user success rate for importing USAA tax info into TurboTax
  • Decreased user time spent on task
  • Opportunity for further partnership between Intuit and USAA

Overall impact

  • Widget integration included experiences enabling document upload and import
  • Widget also integrated in ProTax Online, Intuit's professional tax accountant software
  • Established principles and patterns for auth field help across all widget flows
  • Featured robust error handling for 1099 import in TurboTax, increasing error coverage from 0% to 75%
